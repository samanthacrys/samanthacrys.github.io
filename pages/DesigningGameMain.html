When you're new to game designing, you can often feel overwhelmed knowing what to do. You can spend a lot of time researching and looking at what people do, but not necessarily understand what you are doing or why you should do something the way people are saying. This area of the site is to help you learn and understand how designing works without all the difficulty or falling into pitfalls that those new to programming can.

What You Need

The basics that you need are a pencil/pen and paper.

For those that want to do all of their designing on the computer, you will need a Text program and a paint program at a minimum. This means you can use notepad and paint on the PC. If you are using Linux or MacOS, you can use the built in text editor program found there if you do not have a program like Word. For a basic art program on Linux and MacOS, you will need to find something. I use Krita and highly recommend it. You just want something that can do boxes and arrows when doing a text-adventure. If you are someone that is looking specifically for a flowchart program, you can use multiple programs such as Visio, Flowchart Designer, LucidChart, and Draw.io. Have Fun

Since you're unlikely to find a github set up for the design process, what with all the images being available here, I want to just encourage you to have fun and let yourself be free. You're in the designing phase and, while you may find out you may be unable to do something for whatever reason, the fact is that you need to put it down so you know what you're thinking of.
